SQL Data Cleaning Project

Project Overview

This project demonstrates how SQL can be used to clean and prepare raw datasets for analysis. The objective is to improve data quality by identifying inconsistencies, removing duplicate records, handling missing values, and standardizing data formats to ensure accurate and reliable analysis.

Objectives
Remove duplicate records
Handle missing or NULL values
Standardize inconsistent data
Correct formatting issues
Split and transform columns
Improve overall data quality
Prepare data for further analysis and visualization
Tools Used
SQL
MySQL 
GitHub
Dataset

The dataset contains raw records with various quality issues, including duplicate entries, missing values, inconsistent formatting, and unnecessary columns.

Data Cleaning Process
1. Inspect the dataset
Understand the structure
Identify quality issues
2. Remove duplicate records
Use ROW_NUMBER()
Delete repeated entries
3. Standardize data
Trim spaces
Convert text to a consistent case
Fix inconsistent values
4. Handle missing values
Replace NULL values
Fill missing information where applicable
5. Format dates
Convert dates into a standard format
6. Split columns
Separate combined fields into individual columns
7. Remove unnecessary columns
Delete redundant or unused attributes
